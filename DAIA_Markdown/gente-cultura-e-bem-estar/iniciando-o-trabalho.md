# Iniciando o Trabalho

Conforme visto no tópico anterior, para que os itens fiquem disponíveis para trabalhar, é necessário que a estrutura criada seja publicada e os itens de trabalho passem a constar do backlog do projeto.

 ![Figura 1: itens publicados](/api/attachments.redirect?id=6ef7e796-8146-4e92-8e97-09bb07beda79)


:::warning
Uma vez criados, cada item de trabalho possui informações importantes para serem preenchidas além daquelas já informadas quando da criação. Para cada item, **consulte os documentos relacionados abaixo**.

:::

O Jira possui uma característica que possibilita que os itens de trabalho fiquem disponíveis no board para gestão visual, somente quando incluídos em uma sprint ativa.

Conforme demonstrado na imagem acima, uma vez que os itens estão disponíveis no backlog, é possível selecionar e arrastar para uma sprint existente, ou é possível criar uma nova sprint a partir dos novos itens.

Na primeira opção, o Jira sempre irá alertar quanto à mudança do escopo da sprint em andamento, isso porque essa “alteração” irá interferir nos relatórios e indicadores que podem ser gerados quando uma sprint é finalizada no Jira (abordaremos esse tema em outro tópico).

 ![Figura 2: adicionando itens em uma sprint em andamento](/api/attachments.redirect?id=4e5253bc-ab60-4f46-95ba-ac064131fc96)

Na segunda opção, uma nova sprint será criada, possibilitando mover os itens que estão no backog ou até mesmo itens que estejam em outras sprints para a nova sprint criada.

 ![Figura 3: criando uma nova sprint](/api/attachments.redirect?id=9671d584-b5bc-4300-b640-0f5c9fe5cff8)

 ![Figura 4: adicionando itens à nova sprint](/api/attachments.redirect?id=0b2e245b-4068-4d4f-b001-e809905c8f25)

Para ilustrar a gestão visual no board, iremos adicionar os itens na nova sprint criada e iniciar a sprint para que os itens fiquem visíveis para gestão através do board (quadro).

 ![Figura 5: iniciando a nova sprint](/api/attachments.redirect?id=5ef17eb7-1401-4120-a1a7-7747792ebbbc)

Para visualizar os itens no board, selecione aopção “Sprints ativos“ no menu lateral esquerdo:

 ![Figura 6: acessando o board (quadro)](/api/attachments.redirect?id=332d4375-9123-4183-a6de-75b77583aae3)

Note que ao acessar o board, por padrão ele irá mostrar todos os itens de todas as sprints em andamento. Para facilitar a gestão caso queira enxergar somente os itens de sua sprint, você poderá utilizar os filtros rápidos ou selecionar somente a sua sprint na opção “Todos os sprints“:

 ![Figura 7: filtros rápidos e especialização do quadro](/api/attachments.redirect?id=42abc54e-d129-4e33-87a8-c62fedfbe27f)

Isso entendido, todas as informações dos itens de trabalho devidamente preenchidas, tudo está pronto para começar a movimentação dos cards. No tópico seguinte apresentaremos o fluxo de trabalho para entendimento da cadeia de valor utilizada.
