# Validando a Estrutura

Conforme descrito no tópico “Criação da Estrutura“, é desejável que o plano de ação tenha sido validado previamente, no entanto, se isso não aconteceu, ou se por alguma situação adversa o planejamento mudou, o Roteiro Avançado provê algumas possibilidades para que essa validação ou revalidação possa acontecer sem afetar os dados do backlog do projeto ou sprints em andamento.

Por exemplo:

Considerando os itens criados nos tópicos anteriores (Épico 12345 e demais itens), se você navegar no projeto de origem, irá perceber que esses itens não existem no backlog do projeto:

 ![Figura 1: navegando no projeto de origem](/api/attachments.redirect?id=ed944055-a8f9-4b2b-8d2e-a07646f8e92a)

Isso ocorre porque o roteiro criado não foi publicado, ou seja, é uma medida de segurança para que os novos itens sejam passíveis de execução somente quando devidamente planejados e validados.

Para esse quesito, o Roteiro Avançado permite que todas as suas alterações sejam revisadas antes de uma publicação:

 ![Figura 2: revisando itens criados para publicação](/api/attachments.redirect?id=b4d630ff-57ee-4762-b5ea-8349b4657cf1)

Durante a revisão, caso algum item não esteja de acordo, ou não vá ser publicado, a gestão é bastante simples. É possível expandir as informações que foram criadas ou alteradas, desmarcar itens que não serão publicados, descartar tudo ou enfim, salvar as suas alterações.

 ![Figura 3: possibilidades na revisão dos itens](/api/attachments.redirect?id=79fc600a-1beb-4383-af00-723d708f2729)

Somente após a publicação, os novos itens passarão a constar realmente no backlog de seu projeto.

 ![Figura 4: itens publicados](/api/attachments.redirect?id=74360129-ded4-4603-a1ad-b761e7db7a5f)
